# Documentation Index

Welcome to the documentation. This index provides an overview of the purpose of each top-level document and directory.

## Core Documents

* [**PRODUCT_REQUIREMENTS.md**](PRODUCT_REQUIREMENTS.md): High-level business requirements, user stories, and feature definitions.
* [**SYSTEM_ARCHITECTURE.md**](SYSTEM_ARCHITECTURE.md): The overall system design, high-level components, and their interactions.
* [**IMPLEMENTATION_ROADMAP.md**](IMPLEMENTATION_ROADMAP.md): High-level timeline, goals, and phases for the project's delivery.
* [**DEVELOPMENT_RULES.md**](DEVELOPMENT_RULES.md): Coding standards, guidelines, and best practices for the project.
* [**AGENTS.md**](AGENTS.md): Information regarding AI agent configurations, roles, and instructions.

## Directories

* [**architecture/**](architecture/): Deep dives into the design and functionality of specific system engines and core components.
* [**algorithms/**](algorithms/): Explanations of core logic, data processing steps, and mathematical models used in the system.
* [**data-model/**](data-model/): Definitions of database schemas, graph models, and state representations.
* [**api/**](api/): Documentation for external and internal interfaces, including CLI commands and event contracts.
* [**implementation/**](implementation/): Detailed, phase-by-phase execution plans and milestone tracking.
* [**testing/**](testing/): Strategy and approaches for unit, integration, and end-to-end testing, as well as benchmarking.
* [**prompts/**](prompts/): Templates and guidelines for LLM interactions used across different system functions.
* [**diagrams/**](diagrams/): Visual representations of system architecture, data flows, and processes.
* [**decisions/**](decisions/): Architectural Decision Records (ADRs) that track major technical choices and their rationale.
* [**glossary/**](glossary/): Definitions of domain-specific terminology and common abbreviations used throughout the project.
